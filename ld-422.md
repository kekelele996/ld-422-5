## 项目主要需求

### 项目概述

【全栈Web应用】科学研究实验记录与协作平台。面向实验室研究团队，支持实验项目管理、实验记录撰写（富文本+数据附件）、试剂库存管理和实验协作审批。**本项目用于验证模型跨文件协同改动能力，禁止将多个职责合并进单一文件。**

### 核心实体（≥ 3 个，每个贯穿全栈）

1. **研究项目（ResearchProject）** — 字段：项目名称、项目编号、项目负责人ID、研究方向（Biology/Chemistry/Physics/Materials/CS/Other）、立项日期、预计结题日期、实际结题日期、项目状态（Proposal/Active/Suspended/Completed/Archived）、经费总额、已用经费
   - 关联：1个项目 → 多个实验记录、多个试剂采购
   - **贯穿全栈**

2. **实验记录（ExperimentRecord）** — 字段：所属项目ID、实验标题、实验目的、实验方法、实验步骤（富文本JSON）、实验结论、实验日期、实验人ID、审核人ID、审核状态（Draft/Submitted/Approved/Rejected/RevisionRequired）、审核意见、附件URL列表
   - 业务动作：创建记录、提交审核、审核通过/驳回/要求修改
   - **贯穿全栈**

3. **试剂（Reagent）** — 字段：试剂名称、CAS号、分子式、纯度、危险等级（Safe/Irritant/Corrosive/Flammable/Toxic/Explosive）、存储条件（RoomTemp/Refrigerated/Frozen/Dark/Ventilated）、供应商、当前库存量、单位（g/mL/L/mol/瓶）、最低库存阈值、存放位置
   - 业务动作：入库、领用、库存预警
   - **贯穿全栈**

4. **试剂领用（ReagentUsage）** — 字段：试剂ID、领用人ID、领用数量、领用日期、所属实验ID、用途说明、审批人ID
   - **贯穿全栈**

5. **项目成员（ProjectMember）** — 字段：项目ID、用户ID、角色（PI/SubPI/Researcher/Student/Assistant）、加入日期
   - **贯穿全栈**

### 核心页面（≥ 4 个）

1. **项目总览** `/dashboard` — 消费：ResearchProject + ExperimentRecord + Reagent
   - 在研项目列表（卡片式）、近7天实验记录数、试剂库存预警列表、待审核实验记录数
   - 使用 `<StatusBadge>`, `<StatCard>`, `<AlertBanner>`

2. **项目管理** `/projects` — 消费：ResearchProject + ProjectMember + ExperimentRecord
   - 项目列表（按状态筛选），项目详情（成员列表+实验记录时间线+经费使用进度条）
   - 使用 `<StatusBadge>`, `<ProgressBar>`, `<Timeline>`

3. **实验记录** `/experiments` — 消费：ExperimentRecord + ResearchProject
   - 实验记录列表（按项目+审核状态筛选），富文本编辑器撰写实验记录（支持插入表格、图片、公式），审核流程
   - 使用 `<StatusBadge>`, `<StepIndicator>`

4. **试剂管理** `/reagents` — 消费：Reagent + ReagentUsage
   - 试剂库存列表（低于阈值红色高亮），试剂详情（领用历史+库存趋势），领用登记
   - 使用 `<StatusBadge>`, `<StockAlert>`, `<EmptyState>`

5. **领用记录** `/reagent-usages` — 消费：ReagentUsage + Reagent
   - 领用流水列表（按人员/试剂/项目筛选），创建领用单（选试剂→填用量→关联实验）
   - 使用 `<StatusBadge>`, `<EmptyState>`

### 横切关注点（≥ 2 个）

1. **RBAC 权限控制**
   - 角色：Admin / PI / SubPI / Researcher / Student
   - 触达文件层：DB role表 + RBAC中间件 + 路由守卫 + 按钮显隐（Student不能审核实验记录）+ 类型定义

2. **操作日志**
   - 记录实验提交/审核、试剂领用、项目状态变更
   - 触达文件层：DB audit_log表 + 日志中间件 + service埋点 + 前端操作日志页 + 类型定义

3. **全局异常处理**
   - 触达文件层：后端异常过滤器 + controller异常 + 前端ErrorBoundary + API拦截器 + 错误码枚举

### 共享枚举/常量（≥ 2 组）

1. **项目状态 ProjectStatus**：Proposal / Active / Suspended / Completed / Archived
   - 后端：`backend/src/types/enums.ts`；前端：`frontend/src/types/enums.ts`

2. **审核状态 ReviewStatus**：Draft / Submitted / Approved / Rejected / RevisionRequired

3. **危险等级 HazardLevel**：Safe / Irritant / Corrosive / Flammable / Toxic / Explosive

4. **存储条件 StorageCondition**：RoomTemp / Refrigerated / Frozen / Dark / Ventilated

### 共享前端组件清单（≥ 3 个）

1. **`<StatusBadge>`** — 所有页面
2. **`<StockAlert>`** — 仪表盘、试剂管理（低库存预警）
3. **`<ProgressBar>`** — 项目管理（经费使用率）、项目详情
4. **`<Timeline>`** — 项目详情（实验记录时间线）
5. **`<StepIndicator>`** — 实验记录（撰写→提交→审核）

### 共享 hooks/utils 清单（≥ 2 个）

1. **`useExperiment`** — 封装实验记录的创建、编辑、提交审核逻辑
2. **`usePagination`** — 通用分页
3. **`formatHazardLevel`** — 根据危险等级返回对应颜色和图标常量

### 后端中间件清单（≥ 2 个）

1. **`authMiddleware`** — JWT 鉴权
2. **`rbacMiddleware`** — 角色校验
3. **`auditLogMiddleware`** — 操作日志
4. **`errorHandlerMiddleware`** — 统一异常
5. **`requestLoggerMiddleware`** — 请求日志

### 技术要求

- **英文短名（用于 Docker Compose）**：`research-lab`
- **前端端口**：18802 / 80
- **后端端口**：19302 / 3000
- **前端**：Vue 3 + TypeScript + Vite
- **UI 库**：Element Plus
- **富文本编辑器**：Tiptap
- **图表**：ECharts
- **后端**：Express + TypeScript
- **数据库**：PostgreSQL 15
- **ORM**：Prisma
- **认证**：JWT
- **状态管理**：Pinia
- **构建工具**：Vite

### 部署要求

Docker Compose 一键部署。
- 顶层 `name: research-lab`
- 不写 `version:` 字段
- PostgreSQL 命名卷 + healthcheck
- 后端 depends_on service_healthy
- 前端 `${FRONTEND_PORT:-80}:80`，后端 `${BACKEND_PORT:-3000}:3000`

`git init` + `.gitignore`。
